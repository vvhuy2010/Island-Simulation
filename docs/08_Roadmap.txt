-----------------------------------------------------------
Development Principles
-----------------------------------------------------------

1.

Code only after
the design is stable.

----------------------------------------

2.

Keep every module
independent.

----------------------------------------

3.

Prefer emergence
over hardcoded behavior.

----------------------------------------

4.

LLM is expensive.

Call it only
when necessary.

----------------------------------------

5.

Every new feature
must support
the Vision.

Otherwise,
move it to a future version.

-----------------------------------------------------------
Long-term Dream
-----------------------------------------------------------

One day,

people will stop asking

"How does this AI work?"

and start asking

"What happened to them?"

At that moment,

Island Simulation
will no longer be
just an AI project.

It will become
a living world.
